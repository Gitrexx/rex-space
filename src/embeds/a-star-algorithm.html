A* Search

The cleanest example of informed search: it's Dijkstra's algorithm with a sense of direction. Get the f = g + h idea and the admissibility condition, and you hold the mental model behind most planning, routing, and heuristic-search systems.

Full mode — deep-dive blocks visible

The one idea

Explore the frontier node that looks most promising overall — where "overall" means the cost already spent to reach it plus an estimate of the cost still to go:

f(n) = g(n) + h(n)

  • g(n) — the known cost of the best path found so far from the start to n.
  • h(n) — a guess of the remaining cost from n to the goal (the heuristic).
  • f(n) — the estimated total cost of a path through n. Always expand the smallest f.
Key insight

A* is a dial between two familiar algorithms. Set h = 0 and you get Dijkstra (uninformed, explores in all directions). Ignore g and follow h alone and you get greedy best-first (fast but not optimal). A* keeps both terms, which is exactly what buys it "fast and optimal" — under one condition on h we'll get to.

How it works

A* keeps a priority queue of frontier nodes (the open set), ordered by f. Each iteration it pops the lowest-f node, marks it done (the closed set), and relaxes its neighbours — updating a neighbour's g whenever it just found a cheaper way to reach it.

  1. Put the start in the open set with g = 0, f = h(start).
  2. Pop the node with the smallest f. If it's the goal — reconstruct the path and stop.
  3. For each neighbour, compute tentative_g = g(current) + cost(current, neighbour).
  4. If that's better than the neighbour's recorded g, record the shorter path (came_from), update g and f, and make sure it's in the open set.
  5. Repeat until the goal pops or the open set empties (no path).
Why pop the goal, not "arrive at" it

A* stops when the goal is removed from the open set, not when it's first discovered. That ordering is the whole point: nothing left in the frontier can beat the goal's f, so the path that reached it is provably the cheapest.

Deep dive — the admissibility & consistency conditions

Admissible heuristic: h(n) ≤ h*(n) for every node, where h* is the true remaining cost. In words, h never overestimates. An admissible h guarantees A* returns an optimal path in tree search.

Consistent (monotone) heuristic: for every edge n → n' with cost c, h(n) ≤ c + h(n') — a triangle inequality on the heuristic. Consistency implies admissibility and, crucially, guarantees optimality in graph search (with a closed set) because once a node is closed its g is already optimal, so you never need to reopen it.

Optimality sketch: suppose A* pops the goal with cost g(goal) = C but a cheaper path of cost C* < C exists. Some node n on that cheaper path is still in the open set. By admissibility, f(n) = g(n) + h(n) ≤ g(n) + h*(n) = C* < C = f(goal). So n would have been popped before the goal — contradiction. Hence C = C*.

Where people go wrong

      
Pitfall · wrong heuristic for the move set
        
Manhattan distance is admissible for 4-connected grids but overestimates once you allow diagonal
          moves — use Chebyshev (or octile) distance there, or you'll silently lose optimality.

      
Pitfall · forgetting to relax
        
If you mark a node visited the first time you see it rather than the first time you expand
          it, you can lock in a longer path. A* must be free to lower a node's g when a cheaper route appears.

      
Pitfall · float ties & stale queue entries
        
Standard heaps don't support decrease-key, so the common trick is to push duplicates and skip stale pops
          (the g > g_best check in the code above). Also add a tie-breaker on h to avoid
          exploring huge equal-f plateaus.
