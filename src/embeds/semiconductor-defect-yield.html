半导体制造 · Defect & Yield 入门

半导体制造流程与缺陷 · 良率分析
Wafer Fab Process · Defect · Yield — Line Engineer 上手指南

你是 Defect team 的 Line Engineer。当一批 wafer 良率（yield）掉下来，你要能看懂：这个 defect 是从哪个 process 引入的？该找哪个部门？这份材料把整个晶圆制造流程、每一步会带进什么缺陷、以及如何从 low yield 反推到源头，一次讲清楚。中文讲解，专有名词都附英文——因为你上工用的是英文。

详细模式——deep-dive 展开中（切到「精简」只看主干）

一个核心心智模型

把晶圆制造想成一层一层盖楼：在一片硅片（wafer）上，重复几百次「铺一层材料 → 印图案 → 刻出结构 → 洗干净」的循环，最后叠出上百亿颗晶体管。每片 wafer 上切成很多个芯片（die），每个 die 就是一颗最终卖出去的芯片。

Defect（缺陷）就是在盖楼过程里混进来的、不该有的东西或不该有的形状：一颗掉下来的微粒（particle）、一道刮伤（scratch）、一处没洗掉的残留（residue）、一个该断开却连上的桥接（bridge）。有些 defect 会让 die 直接失效（killer defect），有些无伤大雅（nuisance defect）。

最该记住的一句

每个 defect 都有来源（source / root cause）和特征（signature）。你作为 Defect team 的工作，本质就是：看 signature，反推 source——从 wafer 上缺陷的样子、位置、分布，倒推出是哪个 process、哪台机台、哪个部门出了问题。这份材料的每一节，都是在给你补齐这条「特征 → 来源」的推理链。

为什么 Line Engineer 也得懂整条 process？因为良率是全流程的连乘：任何一层出问题，最终 die 都可能报废。你要能在几百道工序里定位到那一道，就必须对每一段「做什么、会坏在哪」都有画面感。

三大阶段：FEOL → MEOL → BEOL

整条流程按「盖到哪一层」分成三段，追 defect 时第一步往往就是先判断它属于哪一段：

阶段全称盖什么典型工序
前段FEOL
Front-End-of-Line		做出「元件本体」——晶体管（transistor）、二极管等，直接刻在硅里。氧化、离子注入、Gate 形成、扩散/退火、浅沟槽隔离（STI）
中段MEOL
Middle-End-of-Line		做「接点」——把 FEOL 的元件接出来（contact / local interconnect）。Contact 蚀刻、钨（W）填充、接触层 CMP
后段BEOL
Back-End-of-Line		盖「金属大楼」——多层金属线（metal）+ 介电层（dielectric），把元件连成电路。金属沉积、金属/介电层蚀刻、Via、Damascene、金属 CMP
快速判读

看到短路 / 桥接（bridge）、金属线断裂（open）之类，先想 BEOL（后段金属）；看到漏电、临界电压偏移、结（junction）异常，先想 FEOL（前段元件）。这一步「归段」能立刻把嫌疑部门砍掉一大半。

Deep dive — 一层金属是怎么盖出来的（Damascene）

先进制程的 BEOL 金属线多用镶嵌工艺（Damascene）而不是「铺金属再刻」：先在介电层（dielectric）里刻出沟槽（trench）和 via，再填铜（Cu electroplating），最后用 CMP 把多余的铜磨平、只留沟槽里的铜线。所以在 BEOL，很多致命缺陷来自这三步的交界：填不满 → 空洞（void，造成 open）；磨过头 → 碟陷（dishing）；磨不够 → 铜残留（Cu residue，造成 bridge/short）。记住这个「刻槽→填铜→磨平」三拍子，后面看 signature 会更有画面。

测验 1：CP（探针）测出很多 die 相邻的两条金属线短路（bridge / short）。你第一时间该往哪个阶段查？

  • FEOL——因为晶体管本体在那里做
  • BEOL——金属互连层在后段，短路多半是金属/介电层蚀刻或金属 CMP 没磨干净
  • MEOL——接触层不会造成短路
金属线之间的桥接是典型的 BEOL 问题：金属蚀刻残留、金属 CMP 铜残留、或介电层填充不良都会让本该分开的两条线连上。FEOL 的问题更多表现为电性异常（漏电、Vt 偏移）而非金属短路。先「归段」能省下大量排查时间。

单元制程（Unit Process）与它会带进的缺陷

整条流程由少数几种「单元制程」反复组合而成。把每一种「做什么 + 会坏在哪」记熟，你就有了缺陷来源的地图。下面按「铺 → 印 → 刻 → 掺 → 洗 → 平」来记。

① 薄膜沉积 Thin-Film Deposition

在 wafer 上「铺一层材料」——氧化层、氮化层、金属、介电层。主要方式：CVD（化学气相沉积）、PVD（物理气相沉积 / 溅镀）、ALD（原子层沉积，一次一原子层，最精准）。

  • 会带进的 defect：颗粒（particle，腔体剥落/气体带入）、厚度不均（造成后续厚薄不一）、针孔（pinhole）、薄膜内部空洞（void）、附着不良（peeling / 剥落）。
  • 归属线索：Thin Films / CVD / PVD 部门；particle 常呈「单腔集中」——同一 chamber 的 wafer 特别脏。

② 光刻 / 微影 Photolithography（Photo / Litho）

把光罩（mask）上的电路图案「印」到 wafer 上：涂光阻（photoresist）→ 曝光（expose，透过 mask 打光）→ 显影（develop，洗掉被光改变的光阻）。这是整条线最关键、最贵、也最容易出系统性缺陷的一步。

  • 会带进的 defect：套刻偏移（overlay，层与层没对准）、关键尺寸偏差（CD, Critical Dimension）、光阻残留/缺损、显影不良、以及最有辨识度的——光罩缺陷造成的「重复性缺陷」（repeating defect）：mask 脏一点，每一个曝光单元（shot / field）的同一位置都会复制一个缺陷。
  • 归属线索：Photo 部门；「每个 shot 同位置重复出现」几乎等号 = 光罩 / reticle 问题——这是你最该条件反射的一条。

③ 蚀刻 Etch

把没有被光阻保护的地方「刻掉」，留下想要的结构。干法（plasma / 电浆）为主，湿法（wet etch，用化学液）为辅。

  • 会带进的 defect：蚀刻残留（etch residue，没刻干净 → 短路）、过蚀刻（over-etch，刻太多 → 断路/open）、聚合物残留（polymer）、微沟槽（micro-trench）、以及被 particle 挡住而没刻掉的桥接（bridge）。
  • 归属线索：Etch 部门；残留造成 bridge、过蚀造成 open。

④ 离子注入 Ion Implantation · 扩散/退火 Diffusion / Anneal

往硅里「掺杂」——打入杂质离子改变导电性（做出 source/drain、well），再高温退火（anneal）修复晶格、激活杂质。属于 FEOL。

  • 会带进的 defect：剂量/能量偏差（造成电性偏移，属 parametric）、注入不均、退火温度不均造成的边缘环状分布（下面 signature 会讲）。
  • 归属线索：Implant / Diffusion（Furnace）部门；这类缺陷常常「看不见」——不是颗粒，而是电性异常，要靠 WAT/CP 电性数据抓。

⑤ 清洗 Clean / Wet Process

每道关键工序前后都要洗——洗掉 particle、残留、金属污染。是缺陷的「清道夫」，但洗不好也会自己带进缺陷。

  • 会带进的 defect：清洗不足（残留 particle/金属污染）、清洗过度或化学品带入的新 particle、水痕（watermark）、刮伤。
  • 归属线索：Wet / Clean 部门；金属污染（metal contamination）造成的良率损失常是它。

⑥ 化学机械研磨 CMP, Chemical-Mechanical Planarization

用研磨液（slurry）+ 抛光垫（pad）把表面「磨平」，让下一层能盖在平整面上（先进制程层数多，不磨平会盖歪）。

  • 会带进的 defect：刮伤（scratch，slurry 里的大颗粒或 pad 划出）、碟陷（dishing）、侵蚀（erosion）、金属残留（磨不够 → bridge）、slurry 残留 particle。
  • 归属线索：CMP 部门；刮伤（尤其是弧形/直线状的划痕）几乎就是 CMP 或搬运（handling）。
一句话对照

Bridge/短路→ 残留没洗/没刻/没磨掉（Etch、CMP、Clean）；Open/断路→ 该连的没连上（填不满 void、过蚀、断线）；重复性缺陷→ 光罩（Photo）；刮伤→ CMP / 搬运；电性偏移→ Implant / Diffusion / Photo（CD）。

测验 2：SEM 复检发现某缺陷在每一个曝光单元（shot）的完全相同位置都出现一个。最可能的根因是？

  • 随机掉落的 particle——多找几片就没了
  • CMP 刮伤——研磨造成
  • 光罩 / reticle 上有缺陷——每次曝光都把它复制到同一相对位置
曝光是「一个 shot 一个 shot」把光罩图案重复印满整片 wafer。如果缺陷在每个 shot 的同一相对位置重复（hard repeater），说明它被「印」了很多次——来源就是那个被复制的东西：光罩（mask/reticle）。随机 particle 不会这么规律，CMP 刮伤是连续线状而非规律点阵。这是 Photo 部门的问题。

灰 = 硅基板（Si）·蓝 = 金属/薄膜·绿 = 光阻（photoresist）·红 = 缺陷。目标是刻出「两条分开的金属线」；缺陷让中间该刻开的缝被挡住，最后两条线连在一起 = 短路。

Deep dive — 为什么一颗小 particle 能变成致命短路

关键在蚀刻是「有保护就留、没保护就刻」。光阻是「计划中的保护」，而 particle 是「计划外的保护」——它盖住了本该被刻掉的金属，蚀刻刻不到那里，去光阻后那块金属就留下来，把两条本应分开的线桥接成短路。所以在沉积/曝光阶段一颗看似无害的颗粒，会被下游的蚀刻「翻译」成一个电性致命缺陷。这也是为什么 Defect team 要往上游追：缺陷「现形」的那一步（蚀刻后被 inline 检测抓到），常常不是它「被引入」的那一步（沉积时掉的颗粒）。

检测与复检流程（inline flow）

  1. 线上检测 Inline Inspection——在关键工序后用光学/电子束机台扫描 wafer，找出缺陷坐标，产出缺陷图（defect map）。抓的是「哪里有异常」，还不知道是什么。
  2. 自动缺陷分类 ADC, Automatic Defect Classification——机器自动把缺陷归类（particle / scratch / pattern…），快速分堆。
  3. 电镜复检 SEM Review——挑代表性缺陷用扫描电镜放大看清楚，确认到底是什么、多大、在哪一层。这一步「确诊」。
  4. 量测 Metrology——量厚度、CD、overlay 等尺寸参数（不是找缺陷，是量「有没有做准」）。
Inline defect vs. 电性 fail

Inline 检测看到的是「物理缺陷」（看得见的颗粒/刮伤）；而 CP/WAT 测到的是「电性失效」（die 能不能用）。你的核心推理是把两者叠在一起：inline defect map 上的缺陷位置，是否对得上 wafer bin map 上 fail 的 die？对得上 → 这类缺陷是 killer；对不上 → 可能是 nuisance，或者 fail 另有其因（电性/参数）。

Deep dive — 中心/边缘/环状为什么对应特定制程

很多 signature 直接反映工艺的物理对称性。晶圆加工里大量步骤是「旋转对称」的：旋涂光阻（spin coating）、CMP（wafer 转 + pad 转）、腔体内气流/温度分布、电镀（plating）。所以当某个参数在半径方向不均匀，缺陷就会呈现中心（center）或同心环 / 边缘环（edge-ring / donut）：中心通常对应旋涂/研磨/气流的中心区，边缘环常对应温度控制（RTP/furnace 边缘散热快）、edge bead、夹持（chuck/clamp）接触或电镀边缘效应。而直线/弧线 = 机械接触（刮伤、搬运）；规律点阵 = 光罩重复；无规律 = 随机 baseline particle。把「形状 ↔ 物理对称性」连起来，你几乎不用查表也能猜个八九不离十。

测验 3：wafer bin map 上，fail 的 die 集中在整圈最外缘一环（edge-ring），中间干净。下面哪个方向最不该优先怀疑？

  • 退火/RTP 温度控制——边缘散热快、温度不均
  • 电镀 / 蚀刻的边缘均匀性（edge effect）
  • 光罩（reticle）缺陷——它造成的是每个 shot 同位置的规律重复，不是整圈边缘环
边缘环（edge-ring）是典型的径向/边缘均匀性问题：温度（RTP/furnace 边缘）、电镀/蚀刻的边缘效应、晶圆夹持接触都会造成。光罩缺陷的 signature 是规律的重复点阵（每个 shot 同位置），空间上和「整圈边缘」完全不同，所以最不该优先怀疑它。形状对不上，就先排除。

良率（Yield）：定义、分类、模型

良率（yield）= 好的 die 数 ÷ 总 die 数。它在不同阶段有不同的量法，追问题时要说清是哪个：

名称英文含义
线上良率Line Yieldwafer 在产线上没被报废（scrap）的比例——你 Line Engineer 天天盯的。
晶圆测试良率CP / Wafer Sort Yield探针测试（Circuit Probe）后，好 die 的比例。缺陷良率损失主要看这里。
成品良率Final Test Yield封装后最终测试的良率。

良率损失的三种类型（先分类再排查）

  • 随机缺陷损失 Random defect——零星散落的 particle 等造成，分布随机，靠降低整体缺陷密度（D0）来改善。
  • 系统性损失 Systematic——有固定 signature（中心/边缘/重复/某腔），对应某个具体根因，抓到就能一次修好一大片。这是你最有价值的猎物。
  • 参数 / 电性损失 Parametric——不是「缺陷」，是尺寸/电性偏了（CD、Vt、厚度），die 长得对但性能不达标。要靠 WAT/SPC 数据抓，不是靠 defect map。

良率模型：为什么大芯片天生良率低

最经典的 Poisson 良率模型：Y = exp(−D0 × A)。其中 D0 是缺陷密度（每 cm² 的致命缺陷数），A 是单个 die 的面积。直觉：die 越大、缺陷密度越高，踩到致命缺陷的概率越大，良率越低。这解释了为什么大芯片（GPU、CPU）良率远低于小芯片。

Deep dive — 为什么真实 fab 用负二项式（缺陷会聚集）

Poisson 假设缺陷「完全随机、互相独立」，但真实缺陷会成堆聚集（clustering）——一次 chamber 事件在局部撒一片。聚集会让实际良率高于 Poisson 预测（坏 die 挤在一起，浪费的「好区域」变少）。所以业界常用负二项式模型（Negative Binomial）：Y = (1 + D0·A/α)^(−α)，其中 α 是聚集系数（cluster factor）：α 越小 = 越聚集；α → ∞ 时退化回 Poisson。下面的代码可以自己调参数看差异。（还有 Murphy、Seeds 等模型，思路都是在 Poisson 上修正缺陷分布。）

由 Y = exp(−D0·A)，良率随面积 A 指数下降。die 越小，A 越小，Y 越高。所以同样的缺陷水平下，小 die 良率更高——这也是为什么评估某缺陷的影响时，不能只看缺陷数，还要看 die 大小和缺陷「杀伤率」。良率和 D0、A 两个都相关，「D0 相同就一样」忽略了面积。

      

    


    
    

      
从 Low Yield 反推：你的核心工作流

      
良率掉了，怎么一步步查到「哪个 process、哪台机、哪个部门」？这是 Defect team 的看家本领。标准套路：

      
    
        
  1. 发现异常 Detect / Excursion——SPC 报警、良率跌破 baseline、或 inline 缺陷数暴增（excursion）。先确认是真掉了，不是量测噪声。
    2. 
        
  2. 看 signature——调出 wafer bin map，判断空间形状（中心/边缘/重复/聚集/随机）。形状先帮你砍掉一半嫌疑部门。
    3. 
        
  3. 叠图关联 Overlay——把 inline defect map 叠到 bin map 上：缺陷位置 ↔ fail die 位置对得上吗？对得上 → 找到 killer 缺陷类型；对不上 → 往电性/参数方向。
    4. 
        
  4. 共通性分析 Commonality Analysis——把出问题的 wafer/lot 拉出来，比对它们的共同经历：同一台机？同一个 chamber？同一段时间？同一 recipe？坏的都走过、好的都没走过的那台机/那一步，就是最大嫌疑。
    5. 
        
  5. SEM 确诊 + 锁定制程——对代表缺陷做 SEM review 看清楚是什么、在哪一层，结合共通性锁定到具体工序 + 机台——那就是负责的部门。
    6. 
        
  6. 纠正 Corrective Action——通知对应部门（保养机台 PM、修 recipe、隔离 quarantine、清 chamber、换光罩…），追踪良率是否恢复。
    7.

测验 5：某关键层 inline 缺陷数（defect count）翻倍上升，但 CP 良率几乎没变。最合理的解释是？

        
    
          
  • 检测机台坏了，数据不可信，直接忽略
    • 
          
  • 新增的多半是 nuisance（无害）缺陷，kill ratio 低，没落在关键结构上
    • 
          
  • 良率一定会跟着掉，只是还没反映出来
    • 
        

        
不是每个缺陷都会杀死 die——这就是杀伤率（kill ratio）的概念。缺陷数涨但良率不动，通常说明新增的是 nuisance defect（落在无关区域、或太小不致命），kill ratio 低。当然仍要 SEM 确认它们是什么、会不会影响可靠度，但「缺陷数 ≠ 良率损失」是核心认知。直接忽略数据是错的（要确认），「良率一定会掉」也把 count 和 yield 划了等号。

      

    


    
    

      
新手最容易踩的坑

      
坑 1 · 缺陷数 = 良率损失
        
Inline defect count 高不代表良率一定差。要看kill ratio（杀伤率）和位置——落在关键结构上的少数缺陷，比落在空白区的一堆缺陷杀伤力大得多。永远把 defect map 和 bin map 叠起来看。

      
坑 2 · 缺陷「现形处」就是「引入处」
        
Inline 检测抓到缺陷的那一步，往往不是它被引入的那一步（如动画所示：沉积时掉的颗粒，蚀刻后才现形为短路）。要往上游追根因，别停在检测点。

      
坑 3 · 不分类型就乱查
        
随机（random）、系统性（systematic）、参数（parametric）三类损失的查法完全不同。参数问题（CD/Vt 偏移）用 defect map 永远查不到——要看 WAT/SPC 电性数据。先分类，再选工具。

      
坑 4 · 忽略边缘 die 天生较低
        
Wafer 最外圈的 die 因边缘效应良率天生偏低，常在 edge exclusion 范围内。别把「正常的边缘偏低」误报成 excursion——要和 baseline 比，不是和「满片完美」比。

      
坑 5 · 共通性只看坏的
        
「坏 wafer 都走过 X 机台」不足以定罪——如果好 wafer 也都走过 X，它只是产能大。一定要做 fail vs pass 对比，找「坏的走过、好的没走过」的那台。
